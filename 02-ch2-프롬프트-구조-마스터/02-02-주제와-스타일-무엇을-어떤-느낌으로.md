# 주제와 스타일 — 무엇을 어떤 느낌으로

> 프롬프트의 핵심 두 축. '무엇을 그릴까'와 '어떤 느낌으로 그릴까'를 마스터합니다.

## 개요

"고양이 그려줘"라고 했을 때, AI는 수천 가지 고양이 중 어떤 걸 그려야 할지 모릅니다. 이번 세션에서는 주제를 **구체적으로** 묘사하는 기법과, 스타일 키워드 하나로 이미지의 세계를 완전히 바꾸는 방법을 배웁니다.

**학습 목표**:
- 주제 구체성 4단계에 따른 결과 차이를 이해한다
- 주요 스타일 키워드를 활용해 원하는 비주얼 톤을 만든다
- 아티스트·시대 레퍼런스를 프롬프트에 효과적으로 활용한다

## 주제 구체성의 4단계

온라인 쇼핑몰에서 "티셔츠"를 검색하면 수만 개가 나오지만, "네이비 린넨 오버핏 반팔 티셔츠"로 검색하면 원하는 제품이 바로 나오죠.

### Level 1 — 단순 명사

```
a fox
```

→ 종, 색상, 환경, 포즈를 전부 AI가 랜덤으로 결정

![Level 1 결과](images/ch02/02-level1-fox.png "Level 1")

### Level 2 — 형용사 + 명사

```
a red fox with a fluffy tail
```

→ 외형은 잡히지만, 어디서 뭘 하는지는 모름

![Level 2 결과](images/ch02/02-level2-fox.png "Level 2")

### Level 3 — 행동 + 환경

```
a red fox exploring a misty autumn forest at dawn
```

→ 장면이 보이기 시작. 안개, 가을, 새벽이 분위기까지 결정

![Level 3 결과](images/ch02/02-level3-fox.png "Level 3")

### Level 4 — 감정 + 디테일

```
a curious red fox cautiously exploring a misty autumn forest at dawn, golden light filtering through maple leaves, dewdrops on its whiskers
```

→ 감정(호기심, 조심스러움)과 미세한 디테일까지. 거의 머릿속 그림 그대로

![Level 4 결과](images/ch02/02-level4-fox.png "Level 4")

### 구체성 체크리스트

| 체크 항목 | 질문 | 예시 |
|-----------|------|------|
| **누구/무엇** | 주인공이 명확한가? | "여우" → "붉은여우" |
| **외형 특징** | 색상, 크기, 질감? | "털이 풍성한", "반짝이는 비늘" |
| **행동/포즈** | 무엇을 하고 있나? | "달리는", "웅크린", "올려다보는" |
| **환경/배경** | 어디에 있나? | "안개 낀 숲속", "네온사인 골목" |
| **소품/디테일** | 장면을 풍부하게 할 요소? | "옆에 놓인 오래된 책", "떨어지는 꽃잎" |

> ⚠️ "정말 아름다운 고양이"처럼 감탄사만 나열하면 AI에게 별 도움이 안 돼요. **시각적으로 구분 가능한** 정보를 넣으세요.

## 스타일 키워드 — 한 단어가 세계를 바꾼다

같은 노래도 재즈 편곡과 록 편곡이 완전히 다르듯, 같은 주제에 다른 스타일 키워드를 붙이면 전혀 다른 이미지가 됩니다.

### 매체 기반 스타일 — "어떤 재료로 그린 것처럼"

| 키워드 | 효과 | 어울리는 주제 |
|--------|------|--------------|
| `oil painting` | 두껍고 풍부한 질감 | 인물, 풍경, 정물 |
| `watercolor` | 부드럽고 몽환적 | 자연, 꽃, 감성 장면 |
| `pencil sketch` | 거칠고 원초적 | 컨셉 아트, 건축 |
| `digital art` | 깔끔한 디지털 질감 | 게임 아트, 일러스트 |
| `3D render` | 입체감 있는 오브젝트 | 제품, 캐릭터 |

**같은 주제에 매체만 바꾼 비교:**

```
a cozy treehouse surrounded by autumn leaves, oil painting style
```

![유화 스타일 트리하우스](images/ch02/02-treehouse-oil.png "유화 트리하우스")

```
a cozy treehouse surrounded by autumn leaves, watercolor style
```

![수채화 스타일 트리하우스](images/ch02/02-treehouse-watercolor.png "수채화 트리하우스")

```
a cozy treehouse surrounded by autumn leaves, 3D render style
```

![3D 렌더 스타일 트리하우스](images/ch02/02-treehouse-3d.png "3D 트리하우스")

### 사조/장르 기반 스타일

| 키워드 | 핵심 특징 | 대표 비주얼 |
|--------|----------|------------|
| `Impressionism` | 빛의 순간, 부드러운 붓터치 | 모네의 수련 |
| `Art Nouveau` | 유기적 곡선, 장식적 | 뮈샤의 포스터 |
| `Cyberpunk` | 네온, 비 젖은 거리 | 블레이드 러너풍 |
| `Surrealism` | 비현실적 조합, 꿈 | 달리의 녹아내리는 시계 |
| `Minimalism` | 단순한 형태, 여백의 미 | 깔끔한 라인 |
| `Vaporwave` | 80-90년대 노스탤지어 | 핑크-보라 그라데이션 |

```

```

![초현실주의 도서관](images/ch02/02-library-surrealism.png "초현실주의 도서관")

```
a grand library with towering bookshelves, Vaporwave aesthetic, pink and purple neon glow, retro digital feel
```

![베이퍼웨이브 도서관](images/ch02/02-library-vaporwave.png "베이퍼웨이브 도서관")

## 팁과 주의사항

> 🔥 **실무 팁**: 프롬프트 작성이 막막할 때는 **역공학**을 추천해요. 마음에 드는 이미지를 ChatGPT에 업로드하고 "이 이미지를 만들려면 어떤 프롬프트를 써야 해?"라고 물어보세요.

> ⚠️ **흔한 오해**: "스타일 키워드를 많이 넣을수록 좋다" — 아닙니다. 핵심 1~2개 + 보조 키워드가 최적이에요.

> 💡 **플랫폼별 차이**: ChatGPT는 "비 온 뒤 수채화처럼 색이 번진 느낌" 같은 서술형이 잘 먹히고, Midjourney는 `oil painting, dramatic lighting, Renaissance` 같은 키워드 나열이 효과적이에요.

## 핵심 정리

| 개념 | 설명 |
|------|------|
| **구체성 4단계** | 명사 → 형용사 → 행동+환경 → 감정+디테일 |
| **매체 기반 스타일** | oil painting, watercolor 등 — "재료"를 바꾸는 키워드 |
| **사조/장르 스타일** | Impressionism, Cyberpunk 등 — "미학 체계"를 불러오는 키워드 |
| **기술적 스타일** | photorealistic, flat design 등 — "렌더링 방식"을 지정 |
| **아티스트 레퍼런스** | 이름 직접 사용, 시대 사용, 특성 풀어쓰기 |
| **스타일 조합** | 1~2개 핵심 + 보조 키워드가 최적 |

## 다음 세션 미리보기

주제와 스타일로 "무엇을 어떤 느낌으로" 정했다면, 다음은 **"어떤 프레임에 담을 것인가"**입니다. `wide shot`, `close-up`, `bird's eye view` 같은 구도 키워드가 이미지의 시선 흐름과 감정을 어떻게 바꾸는지 알아봅니다.
